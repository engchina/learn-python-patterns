# Python 设计模式学习项目 (Python Design Patterns Learning Project)

> **核心思想**: 通过系统化的学习路径和丰富的实践示例，帮助开发者深入理解和掌握 23 种经典设计模式在 Python 中的实现与应用。

## 📚 项目概述

本项目是一个完整的 Python 设计模式学习资源库，包含了面向对象基础知识和 23 种 GoF 设计模式的详细实现。每个模式都提供了从基础概念到实际应用的完整学习路径，特别针对 Python 语言特性进行了优化。

**项目特色**:
- **系统化学习路径**: 按照难度和依赖关系精心安排的学习顺序
- **丰富的代码示例**: 每个模式包含 5-6 个不同复杂度的示例
- **真实世界应用**: 每个模式都包含实际项目中的应用案例
- **Python 特色实现**: 充分利用 Python 语言特性的优雅实现
- **详细文档说明**: 每个模式都有完整的概念解析和使用指南

## 📚 内容结构

### 第一阶段：基础准备
> **核心思想**: 将现实世界的事物抽象为代码中的对象，通过封装、继承和多态来构建可维护、可扩展的软件。

- **01_面向对象基础** - Object-Oriented Programming Basics
  - 类与对象、继承、多态、封装、组合等基础概念
  - 为理解设计模式奠定坚实的OOP基础

### 第二阶段：创建型模式 (Creational Patterns)
> **核心思想**: 关注对象的创建过程，提供了创建对象的最佳方式，将对象的创建与使用分离。

- **02_简单工厂模式** - Simple Factory Pattern
  - 定义一个工厂类，根据参数返回不同类的实例
- **03_工厂方法模式** - Factory Method Pattern
  - 定义创建对象的接口，让子类决定实例化哪个类
- **04_抽象工厂模式** - Abstract Factory Pattern
  - 提供一个创建一系列相关或相互依赖对象的接口
- **05_单例模式** - Singleton Pattern
  - 保证一个类仅有一个实例，并提供一个全局访问点
- **06_建造者模式** - Builder Pattern
  - 将一个复杂对象的构建与它的表示分离
- **07_原型模式** - Prototype Pattern
  - 通过复制现有实例来创建新的实例

### 第三阶段：结构型模式 (Structural Patterns)
> **核心思想**: 关注类和对象的组合，描述如何将类或对象结合在一起形成更大的结构。

- **08_适配器模式** - Adapter Pattern
  - 将一个类的接口转换成客户希望的另一个接口
- **09_桥接模式** - Bridge Pattern
  - 将抽象部分与它的实现部分分离，使它们都可以独立变化
- **10_组合模式** - Composite Pattern
  - 将对象组合成树形结构以表示"部分-整体"的层次结构
- **11_装饰器模式** - Decorator Pattern
  - 动态地给一个对象添加一些额外的职责
- **12_外观模式** - Facade Pattern
  - 为子系统中的一组接口提供一个一致的界面
- **13_享元模式** - Flyweight Pattern
  - 运用共享技术有效地支持大量细粒度的对象
- **14_代理模式** - Proxy Pattern
  - 为其他对象提供一种代理以控制对这个对象的访问

### 第四阶段：行为型模式 (Behavioral Patterns)
> **核心思想**: 关注对象之间的通信和职责分配，描述对象之间怎样相互协作。

- **15_责任链模式** - Chain of Responsibility Pattern
  - 使多个对象都有机会处理请求，避免发送者和接收者之间的耦合
- **16_命令模式** - Command Pattern
  - 将请求封装为对象，从而可用不同的请求对客户进行参数化
- **17_解释器模式** - Interpreter Pattern
  - 给定一个语言，定义它的文法的一种表示
- **18_迭代器模式** - Iterator Pattern
  - 提供一种方法顺序访问一个聚合对象中的各个元素
- **19_中介者模式** - Mediator Pattern
  - 用一个中介对象来封装一系列的对象交互
- **20_备忘录模式** - Memento Pattern
  - 在不破坏封装性的前提下，捕获一个对象的内部状态
- **21_观察者模式** - Observer Pattern
  - 定义对象间的一种一对多的依赖关系
- **22_状态模式** - State Pattern
  - 允许一个对象在其内部状态改变时改变它的行为
- **23_策略模式** - Strategy Pattern
  - 定义一系列的算法，把它们一个个封装起来
- **24_模板方法模式** - Template Method Pattern
  - 定义一个操作中的算法的骨架
- **25_访问者模式** - Visitor Pattern
  - 表示一个作用于某对象结构中的各元素的操作

## 🚀 如何运行

### 环境准备
```bash
# 克隆项目
git clone <repository-url>
cd learn-python-patterns

# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate     # Windows
```

### 运行示例
```bash
# 运行面向对象基础示例
python 01_面向对象基础/01_basic_classes.py

# 运行简单工厂模式示例
python 02_简单工厂模式/01_basic_factory.py

# 运行观察者模式示例
python 21_观察者模式/01_basic_observer.py

# 运行任意模式的所有示例
cd 05_单例模式
python 01_basic_singleton.py
python 02_decorator_singleton.py
# ... 依此类推
```

2. **创建型模式**: 02-07 文件夹
3. **结构型模式**: 08-14 文件夹
